# Content Guidelines

The Oyagaa-Oomo Network website should be source-first, careful, and credible. It should help visitors examine the OOMO/UMMO dossier without asking them to believe, join, or accept conclusions.

## Source Hierarchy

Content should be labeled according to this hierarchy:

1. Primary source / original document
2. Scan or archive reference
3. Translation
4. Summary
5. Analysis
6. Hypothesis
7. OON interpretation

These layers should not be blurred. A hypothesis should never be presented as a source. An OON interpretation should never be presented as a document claim.

## Quotes

Quote cards should be handled with special care.

Each quote should eventually include:

- Exact quote text
- Source document ID or title
- Source date, if verified
- Archive or scan reference, if available
- Translation status
- Short context note
- Status label, such as verified, needs review, translation pending, or reference missing

Do not invent quote text, document references, dates, or archive links. If a piece of information is missing, use a TODO placeholder.

Example:

```text
TODO: Add verified source document ID.
```

## Document Cards

Document cards should make the status of each item clear.

Each document card should eventually identify:

- Document title or ID
- Source type
- Date, if verified
- Language
- Translation availability
- Archive link
- Short neutral description
- Related themes
- Review status

Descriptions should summarize what the source material contains, not what visitors should conclude from it.

## Cautious Language

Use language that keeps the distinction between evidence, interpretation, and open questions clear.

Preferred phrases:

- "the documents state..."
- "the corpus describes..."
- "according to the source material..."
- "the source material appears to..."
- "OON interprets this as..."
- "this remains an open question..."
- "one possible reading is..."

Avoid:

- "this proves..."
- "the truth is..."
- "evidence confirms..."
- "they warned us..."
- "hidden knowledge..."
- "disclosure is coming..."
- "you must understand..."

## Protocol Handling

If Collective Practices, Protocol, or related material appears on the site, it should be presented carefully and secondarily.

Protocol content should:

- Be clearly separated from source documents
- Avoid recruitment language
- Avoid spiritual authority language
- Avoid presenting participation as required
- Avoid implying certainty or special access
- Be framed as OON practice, interpretation, or experiment where appropriate

## What Not To Claim

Do not claim that:

- The dossier is proven true
- The documents have a single settled interpretation
- OON replaces existing archives
- OON has final authority over the material
- Visitors should believe before examining sources
- AI analysis can verify claims by itself
- Any unverified quote, date, translation, or archive reference is confirmed

When in doubt, slow down, label the content status, and preserve the open question.
