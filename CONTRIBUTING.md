# Contributing

This project will grow through small, careful contributions. You do not need to be a developer to help.

## How To Suggest Changes

The preferred ways to suggest changes are:

- Open an issue for ideas, corrections, missing sources, or questions.
- Open a pull request for specific edits.
- Keep each contribution focused on one clear purpose.

Examples of focused contributions:

- Add one verified source reference.
- Improve one page of copy.
- Add one group of quote cards.
- Fix one design issue.
- Update one implementation detail.

## Keep Changes Separate

When possible, separate:

- Content changes
- Design changes
- Code changes
- Source/reference additions

This makes review easier and helps avoid accidental changes to unrelated parts of the project.

## Source Additions

Any new source, quote, document, archive link, or translation should include reference information and a status label.

Useful status labels include:

- Verified
- Needs review
- Translation pending
- Reference missing
- Archive link needed
- Interpretation only

If reference information is missing, do not guess. Add a TODO placeholder and describe what needs to be verified.

## Pull Request Expectations

A good pull request should explain:

- What changed
- Why it changed
- What Ivona should review
- Whether any source references still need verification

Pull requests should avoid broad rewrites unless they were discussed first.

## Project Tone

Contributions should keep the website calm, careful, and credible.

Avoid sensational language, overclaiming, cultish tone, recruitment framing, and fear-based disclosure language.
