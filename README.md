# Oyagaa-Oomo Network Website

This repository will host the public website for the Oyagaa-Oomo Network, also known as OON.

The site will become a modern, source-first entry point for the OOMO/UMMO dossier. It is not intended to replace existing archives. Instead, it should help visitors understand what the dossier is, why it may deserve careful examination, and where to go next.

## Current Phase

The project is currently in setup.

This means the repository is being prepared with clear instructions, project context, content standards, and a future implementation plan. The website itself has not been built yet.

At this stage, there is no Astro project, no Tailwind setup, no pages, and no installed dependencies.

## Planned Site Structure

The future website may include:

- Home
- Start Here
- Essential Documents
- Themes
- Watch
- AI Research Kit
- Archives / Resources
- About OON
- Collective Practices / Protocol

The final structure may change as the project develops, but the main goal should remain the same: help skeptical but curious visitors orient themselves quickly and move toward reliable source material.

## Project Direction

The preferred future technical direction is:

- Static website
- GitHub Pages hosting
- Astro + Tailwind CSS
- Markdown or JSON content
- No backend
- No database
- No authentication
- No paid hosting dependency

Future structured content may live in files such as:

- `content/quotes.json`
- `content/videos.json`
- `content/archives.json`
- `content/documents.json`
- `content/ai-prompts.json`

## How This Project Will Evolve

The project should grow in small, reviewable steps.

First, the repository gets its instructions and planning files. Next, a static website scaffold can be added. After that, the homepage and content structure can be built gradually. Real source material should be integrated only when references, status labels, and source boundaries are clear.

Ivona should not need to manually create or configure project files. Codex and future collaborators should do the setup work in small pull requests that are easy to review.
