# Implementation Plan

This plan describes future phases for the Oyagaa-Oomo Network website. The current task only covers Phase 0.

## Phase 0: Repo Setup

Codex should:

- Create repository instruction files.
- Capture project context.
- Define content guidelines.
- Define a future implementation plan.
- Avoid scaffolding the website.
- Avoid installing dependencies.

Ivona should review:

- Whether the project goals are described correctly.
- Whether the tone and credibility principles feel right.
- Whether the future phases match the intended direction.

## Phase 1: Static Site Scaffold

Codex should:

- Scaffold a minimal static Astro project only after explicit approval.
- Keep the setup simple and compatible with GitHub Pages.
- Add Tailwind CSS only if explicitly requested for this phase.
- Add basic project scripts and configuration.
- Avoid building real pages beyond what is necessary to confirm the scaffold works.

Ivona should review:

- Whether the project structure feels understandable.
- Whether the setup remains simple.
- Whether the next phase is clear.

## Phase 2: Homepage MVP

Codex should:

- Build a first homepage draft.
- Focus on orientation, credibility, and clear next steps.
- Avoid presenting unverified claims.
- Use placeholder content where source data is missing.
- Keep the design calm, modern, and non-sensational.

Ivona should review:

- First impression
- Tone
- Page hierarchy
- Visitor journey
- Whether the page answers the 8-second and 60-second goals

## Phase 3: Content Data Structure

Codex should:

- Create structured content files for future quotes, videos, documents, archives, and AI prompts.
- Define clear fields for source status and review status.
- Use TODO placeholders when references are missing.
- Avoid inventing source details.

Ivona should review:

- Whether the content structure matches how she wants to curate the material.
- Whether the status labels are useful.
- Whether adding future content feels manageable.

## Phase 4: GitHub Pages Deployment

Codex should:

- Configure GitHub Pages deployment.
- Keep the site static.
- Confirm build output works with GitHub Pages.
- Document the publishing flow in plain language.

Ivona should review:

- Whether the live site loads correctly.
- Whether the deployment steps are understandable.
- Whether any public-facing placeholder content needs adjustment.

## Phase 5: Visual Refinement

Codex should:

- Refine typography, spacing, layout, navigation, and responsive behavior.
- Preserve the established visual direction.
- Avoid sensational UFO styling.
- Keep the experience credible and source-first.

Ivona should review:

- Visual tone
- Readability
- Mobile layout
- Navigation clarity
- Whether the site feels like OON

## Phase 6: Real Content Integration

Codex should:

- Add real quotes, documents, videos, archives, and AI prompts from verified or clearly labeled sources.
- Preserve the distinction between source, translation, analysis, hypothesis, and OON interpretation.
- Add status labels for anything incomplete.
- Keep each content addition reviewable.

Ivona should review:

- Source accuracy
- Tone
- Content grouping
- Missing references
- Whether interpretation is clearly labeled

## Phase 7: Collaborator Workflow

Codex should:

- Improve contribution instructions as real collaboration begins.
- Add issue or pull request templates if useful.
- Define review paths for content, design, and code.
- Keep the workflow friendly for non-developers.

Ivona should review:

- Whether collaborators can understand how to help.
- Whether review responsibilities are clear.
- Whether the workflow protects source quality.
