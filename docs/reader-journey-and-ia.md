# Reader Journey and IA

This note captures the launch information architecture decision for the Oyagaa-Oomo Network website.

## Core Principle

Navigation should describe what the reader is trying to do.

Page-internal structure should describe how the material is organized.

This keeps the public interface calm and simple while still allowing the corpus to become more structured over time.

## Launch Navigation

The launch top navigation should stay small:

- Start
- Read
- Timeline
- Sources

These labels map to reader intent:

- Start: orient me.
- Read: let me enter the material.
- Timeline: show me the arc.
- Sources: let me verify where this points back.

## Read as the Main Entry Point

Read covers the broader source material, not only letters.

The source material may include:

- letters;
- reports;
- phone calls;
- NR communications;
- GR1 communications;
- later accounts or tweets;
- archive notes;
- OON commentary, clearly separated from source material.

Readers should be able to enter through one communication, one topic, one period, or one source path.

## Topics

Topics are a browse mode inside Read, not a top navigation item at launch.

The Topics route may remain available as a support route or future topic-browser route, but its visible language should make clear that any topic list is provisional until the source model is mature.

## Timeline

Timeline is first-class.

It gives the chronological skeleton of the dossier without requiring the reader to begin with archive detail or assessment language.

Timeline entries should distinguish the kind of event being shown:

- communication received;
- researcher/archive event;
- public/media event;
- source-continuity event;
- OON/public orientation event.

Timeline should not resolve authorship, origin, or disclosure questions.

## Sources

Sources is the verification layer.

The archive sites remain the canonical repositories. This website is an intelligibility and reading interface.

If OON later provides curated reading copies or downloadable study versions, each local copy must point back to archive/source paths and keep these layers distinct:

- source status;
- translation status;
- archive/source path;
- summary;
- analysis;
- OON interpretation.

## FAQ

FAQ handles high-friction questions, not top navigation.

FAQ material should help readers understand sensitive or common points of confusion without making verdict language the primary posture.

Important FAQ areas include:

- why the dossier is still studied after Jose Luis Jordan Pena's authorship claim;
- how to approach source-status language;
- how to distinguish OON commentary from source material;
- why the website points back to external archives.

## Support Links

Support links may include:

- About OON;
- FAQ;
- Source Continuity;
- Misuse & False Attribution;
- Watch / Public Channels.

Do not expose internal repository docs as public footer links.
