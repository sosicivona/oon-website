# Content Governance

This document defines editorial rules for OON website content.

## Editorial Posture

The website is source-first, reading-first, and non-sensational.

It should help readers examine the UMMO/OOMO dossier without asking them to believe, join, dismiss, or accept a conclusion.

The reader stays free to read, doubt, compare, and decide.

## What The Site Is

The website is an intelligibility and reading interface.

It can:

- orient new visitors
- structure reading paths
- explain source-status language
- separate source text from analysis
- route readers toward archives and public OON channels
- present evidence questions cautiously

It should not:

- replace archive sites
- present OON as the source
- present OON as final authority
- prove or disprove the dossier
- invent missing source data
- use sensational UFO aesthetics or disclosure pressure

## Source Layering

Keep these layers separate:

1. Source text
2. Scan or archive reference
3. Translation
4. Summary
5. Analysis
6. Hypothesis
7. OON interpretation

Never present a hypothesis as a source. Never present OON interpretation as a document claim.

## Required Status Visibility

Reader pages and document cards should eventually show:

- source location
- source account or source-chain status
- date or chronology status
- original language
- translation status
- archive reference status
- analysis boundary
- related themes or questions

If any detail is missing, use a clear TODO in source files or docs. Do not invent.

Examples:

```text
TODO: Verify archive URL.
TODO: Confirm translation status.
TODO: Add source date only after verification.
```

## Cautious Language

Preferred wording:

- "the corpus describes..."
- "the letters say..."
- "the documents state..."
- "according to the source material..."
- "recognized source account..."
- "source chain..."
- "OON interprets this as..."
- "this remains an open question..."
- "one possible reading is..."

Avoid:

- "this proves..."
- "confirmed..."
- "debunked..."
- "official source account..."
- "the truth is..."
- "disclosure is coming..."
- "they warned us..."
- "you must understand..."

## Recognized Source Account

Use "Recognized source account" only as a source-status label.

Required explanation:

> Recognized by source researchers as part of the later UMMO/OOMO communication chain; this does not ask the reader to accept the origin claim.

Do not use this label as proof of origin.

## Sensitive Topics

Sensitive, disputed, or high-risk topics require extra caution.

Do not feature claims about public health, geopolitical events, forecast dates, economic collapse, or similar topics on the homepage unless a future governance decision explicitly allows it.

These topics may be held as future Evidence and Questions backlog categories only when:

- source chain is visible
- chronology status is visible
- counter-readings are included
- assessment language remains separate from source text

## Essential Documents

Essential Documents should be introduced only after source review and editorial framing are ready.

Future spotlight:

### D1378 — The Mutual Aid Letter

Safe framing:

"D1378 — The Mutual Aid Letter" may be treated as a future Essential Documents spotlight and described as "one of the most sensitive documents discussed in the dossier."

Do not overstate what it proves. Do not use it as homepage sensationalism.
