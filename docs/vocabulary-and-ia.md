# Vocabulary and Information Architecture

This document defines core vocabulary for the Oyagaa-Oomo Network website. Future content and implementation work should use these terms consistently.

## Core Terms

### The UMMO/OOMO dossier

Use "the UMMO/OOMO dossier" for the full historical phenomenon:

- letters
- later communications
- source texts
- archive records
- translations
- analysis
- public research
- contested interpretations
- questions around origin, chronology, authorship, and meaning

The dossier is the broad frame. It is not a single story, a single sighting case, or a single archive.

### The corpus

Use "the corpus" for the body of letters, messages, and source texts inside the dossier.

The corpus is what readers examine directly. It should be presented through source status, language status, translation status, archive context, and analysis boundaries.

### The case

Use "the case" for public or newcomer framing.

"The case" is useful when helping a visitor understand why the dossier is unusual without asking them to adopt a conclusion. It is an orientation word, not a proof claim.

### OON

OON means Oyagaa-Oomo Network.

OON is an interface and network layer. It is not the source and not the archive. It studies, organizes, contextualizes, discusses, and shares material around the dossier.

OON should not be described as having final authority over the source material.

### Archive sites

The archive sites remain the true archives.

This website is an intelligibility and reading interface. It should guide readers toward archive sources and public channels without pretending to replace them.

## Site IA Roles

### Homepage

Role: orientation and preview hub.

The homepage should help a new visitor understand:

- what the dossier is
- why reading comes first
- where to begin
- what themes exist
- how evidence and questions will be handled
- where sources and public channels live

The homepage should not try to prove the dossier or overwhelm visitors with source detail.

### Start Here

Role: first orientation.

This page should answer:

- What is this dossier?
- How should a reader approach it?
- What should they read first?
- What is source status?
- What is not being claimed?

### Read

Role: corpus entry.

This page should introduce reading paths, corpus types, and the anatomy of future reader pages.

It should feel like an entry into the material, not only an explanation of future implementation.

### Themes

Role: worldview-first pathways.

Themes help readers follow repeated topics across the corpus while keeping source text and interpretation separate.

### Evidence and Questions

Role: cautious assessment framework.

This area should map cases for examination without turning them into conclusions.

### Sources

Role: source ecosystem and public channel directory.

This page should distinguish:

- archive sites
- public OON channels
- source-status language
- source-continuity methodology

### Source Continuity

Role: methodology note.

This page explains how source chain, archive location, translation status, and analysis boundaries should be made visible.

It should remain accessible from Sources, Read, and Start Here, but it does not need to be a primary navigation item once a Sources page exists.

## Naming Rules

- Use "the UMMO/OOMO dossier" when referring to the full historical phenomenon.
- Use "the corpus" when referring to letters, messages, and source texts.
- Use "the case" when orienting newcomers.
- Use "OON" for the network/interface layer.
- Do not write as if OON is the archive.
- Do not write as if OON is the source.
- Do not write as if this website replaces archive sites.
